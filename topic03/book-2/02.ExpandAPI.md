+ Extend the API:
~~~ts

    );

~~~
Create the file `lambdas/addMovie.ts` and initialize it with the following code:
~~~ts
import { APIGatewayProxyHandlerV2 } from "aws-lambda";
import { DynamoDBClient } from "@aws-sdk/client-dynamodb";
import { DynamoDBDocumentClient, PutCommand } from "@aws-sdk/lib-dynamodb";

const ddbDocClient = createDDbDocClient();

export const handler: APIGatewayProxyHandlerV2 = async (event, context) => {
  try {
    console.log("[EVENT]", JSON.stringify(event));
    const body = event.body ? JSON.parse(event.body) : undefined;
    if (!body) {
      return {
        statusCode: 500,
        headers: {
          "content-type": "application/json",
        },
        body: JSON.stringify({ message: "Missing request body" }),
      };
    }

    const commandOutput = await ddbDocClient.send(
      new PutCommand({
        TableName: process.env.TABLE_NAME,
        Item: body,
      })
    );
    return {
      statusCode: 201,
      headers: {
        "content-type": "application/json",
      },
      body: JSON.stringify({ message: "Movie added" }),
    };
  } catch (error: any) {
    console.log(JSON.stringify(error));
    return {
      statusCode: 500,
      headers: {
        "content-type": "application/json",
      },
      body: JSON.stringify({ error }),
    };
  }
};

function createDDbDocClient() {
  const ddbClient = new DynamoDBClient({ region: process.env.REGION });
  const marshallOptions = {
    convertEmptyValues: true,
    removeUndefinedValues: true,
    convertClassInstanceToMap: true,
  };
  const unmarshallOptions = {
    wrapNumbers: false,
  };
  const translateConfig = { marshallOptions, unmarshallOptions };
  return DynamoDBDocumentClient.from(ddbClient, translateConfig);
}
~~~
In the above code we write/put a new item into a DynamoDB table by sending a PutItem command to the DynamoDB engine.

###  POST Payload Validation.

There is a significant weakness in the above solution; we don't validate the JSON structure of the POST request payload. The shape of the payload is only known at runtime, while TypeScript type aliases and interface data types work well at compile-time, but disappear during runtime. Also, Typescript cannot help when the data's source is external, i.e. provided by a user or another application. We need extra tooling to solve this problem, for example, the typescript-json-schema CLI and AJV library. The CLI can create JSON Schema documents from TypeScript definitions. The schemas follow the [Draft 7 JSON Schema][json] format, which many applications understand, including AJV. 

Add typescript-json-schema to the project as a dev dependency:
~~~bash
$ npm install -D typescript-json-schema
~~~
Add a new script to `package.json` as follows:
~~~json
  "scripts": {
    ... as before ...
    "schema": "typescript-json-schema --noExtraProps --required --refs false -o ./shared/types.schema.json  \"./shared/types.d.ts\" \"*\""
 },
~~~
The script generates JSON schema documents from our custom TS type/interface definitions in `shared/types.d.ts`. Run the script:
~~~bash
$ npm run schema
~~~
Briefly examine the generated file `shared/types.schema.json`. We can use these schemas at runtime to validate JSON data, e.g., an HTTP POST payload. The AJV package is ideal for this task; add it as a prod dependency:
~~~bash
$ npm i --save ajv 
~~~
Make the following changes to `lambdas/addMovie.ts`:
~~~ts
// ... existing imports ....
import Ajv from "ajv";
import schema from "../shared/types.schema.json";

const ajv = new Ajv();
const isValidMoviePayload = ajv.compile(schema.definitions["Movie"] || {});

const ddbDocClient = createDDbDocClient();

export const handler: APIGatewayProxyHandlerV2 = async (event, context) => {
  try {
    console.log("[EVENT]", JSON.stringify(event));
    const body = event.body ? JSON.parse(event.body) : undefined;
    if (!body) {
      // as before ....
 }
    // NEW
    if (!isValidMoviePayload(body)) {
      return {
        statusCode: 500,
        headers: {
          "content-type": "application/json",
 },
        body: JSON.stringify({
          message: `Incorrect type. Must match the Movie schema`,
          schema: schema.definitions["Movie"],
 }),
 };
 }
    // Unchanged
    const commandOutput = await ddbDocClient.send(
       // as before ......
 );
    // as before .....
~~~
By default, the TypeScript compiler disallows importing JSON files. To override this,  change the bottom of `tsconfig.json` as follows:
~~~json
 "exclude": [
  ... as before ...
 ],
  "include": [
    "shared/*.json"
 ],
}
~~~
The critical line in `lambdas/addMovie.ts` above is:

>const isValidMoviePayload = ajv.compile(schema.definitions["Movie"] || {});

'isValidMoviePayload' is a dynamically generated function to validate a JS object against the Movie schema produced by typescript-json-schema.

###  Typescript Type Guards 
. 
In the `addMovie.ts` lambda, the 'body' variable's type is any (use cntl-k/cntl-i), but it should be Movie after the validation check passes. [Type guards][guards] can narrow a variable's type to the correct type. Create the file `lambdas/guards.ts` and add the following code:
~~~ts
import Ajv from "ajv";
import schema from "../shared/types.schema.json";
import { Movie, MovieCast } from "../shared/types";

const ajv = new Ajv();
const isValidMoviePayload = ajv.compile(schema.definitions["Movie"] || {});

export function isMovie(data: any): data is Movie {
    return (
      typeof data === 'object' &&
      data !== null &&
      isValidMoviePayload(data)
 )
 }
~~~
The `isMovie` function is a type guard; it returns true if data is a Movie object. In `addMovie.ts`:
+ Add an import statement:
~~~ts
import {isMovie} from './guards'
~~~
+ Change the if statement that tests the body variable:
~~~ts
    if (!isMovie(body)) {
      return {
        statusCode: 500,
        headers: {
          "content-type": "application/json",
        },
        body: ... as before ....}),
    };
 } 
~~~
+ Remove all the unused AJV code at the top of the file.

Now use cntl-k/cntl-i to check the type of the body variable in the PutCommand statement; it's a Movie type.
