# Skill: Feature Checklist

## Purpose
Use this checklist before finishing a spec-driven feature.

## Checklist
- Is the user-visible goal complete?
- Are props typed?
- Is async data handled safely?
- Is API code separated from UI code?
- Is repeated layout extracted if appropriate?
- Are relevant existing skills reused?
- Are route paths correct?
- Are loading and error states handled?
- Is the solution understandable to another student?
- Does the feature match the acceptance criteria in the spec?

## Why it matters
It prevents “it mostly works” submissions and keeps the codebase consistent.

## In this course
This is useful for every movie app lab feature.
